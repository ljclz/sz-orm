1//! SZ-ORM PostGIS 扩展
2//!
3//! 提供 PostgreSQL PostGIS 空间几何查询能力，支持三种实现：
4//!
5//! - **内存实现**（`Memory`）：纯 Rust 几何计算，不连接数据库，适用于测试和基准
6//! - **Stub 实现**（`Stub`）：生成 PostGIS SQL 字符串但不执行，适用于调试
7//! - **真实实现**（`RealPg`，需启用 `real-postgis` feature）：通过 tokio-postgres 连接 PostgreSQL
8//!
9//! # 支持的几何类型
10//!
11//! - `Point`：点
12//! - `LineString`：线串
13//! - `Polygon`：多边形（含洞）
14//! - `MultiPoint` / `MultiLineString` / `MultiPolygon`：多几何体
15//!
16//! 所有几何类型携带 SRID（坐标参考系统 ID），默认 WGS84（SRID=4326）。
17//!
18//! # 支持的空间操作
19//!
20//! | 方法 | SQL 等价 | 说明 |
21//! |------|---------|------|
22//! | `st_distance` | `ST_Distance` | 两点距离 |
23//! | `st_contains` | `ST_Contains` | 包含判断 |
24//! | `st_within` | `ST_Within` | 在内部判断 |
25//! | `st_intersects` | `ST_Intersects` | 相交判断 |
26//! | `st_area` | `ST_Area` | 面积计算 |
27//! | `st_length` | `ST_Length` | 长度计算 |
28//! | `st_buffer` | `ST_Buffer` | 缓冲区 |
29//! | `st_union` | `ST_Union` | 几何合并 |
30//! | `add_geometry_column` | `AddGeometryColumn` | 添加几何列 |
31//! | `create_spatial_index` | `CREATE INDEX ... USING GIST` | 空间索引 |
32//!
33//! # 快速入门
34//!
35//! ```rust
36//! use sz_orm_postgis::{PostgisBuilder, PostgisExt, PostgisProvider, Geometry, Point};
37//!
38//! # #[tokio::main]
39//! # async fn main() -> Result<(), Box<dyn std::error::Error>> {
40//! let wrapper = PostgisBuilder::new(PostgisProvider::Memory).build()?;
41//!
42//! let beijing = Geometry::Point(Point::new(116.404, 39.915));
43//! let shanghai = Geometry::Point(Point::new(121.474, 31.230));
44//!
45//! let distance = wrapper.st_distance(&beijing, &shanghai).await?;
46//! println!("distance: {:.2} m", distance);
47//! # Ok(())
48//! # }
49//! ```
