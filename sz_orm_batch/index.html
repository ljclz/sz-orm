§SZ-ORM Batch — 批量操作

提供批量插入、更新与 UPSERT 能力，支持多值 INSERT、CASE WHEN UPDATE 与分片感知的批量执行，并返回生成的 SQL 供审计。

§主要类型

  • BatchResult — 批量操作结果
  • BatchOperations trait — 批量操作接口

Structs§

BatchProgress
批量操作进度信息，传递给进度回调。
BatchResult
批量操作结果。generated_sqls 持有实际生成的 SQL 语句，供调用方执行与审计。
ChunkProcessResult
分块处理结果，记录每个分块的处理情况。
DefaultBatchOps
默认批量操作实现。生成多值 INSERT、CASE WHEN UPDATE、ON CONFLICT/ON DUPLICATE UPSERT。
UpsertResult
带冲突目标的 UPSERT 结果。

Enums§

BatchStage
批量操作阶段，用于进度回调报告。
ConflictTarget
UPSERT 冲突目标（ON CONFLICT 子句的目标）。
RollbackStrategy
批量操作回滚策略。
UpsertMode
Upsert 语法模式：MySQL 风格（ON DUPLICATE KEY UPDATE）或 PostgreSQL 风格（ON CONFLICT DO UPDATE）。
