pub fn register_table(&self, name: impl Into<String>, rows: Vec<Value>)

Registers (or replaces) the rows for a virtual table that will be included in the next Self::backup / Self::export_sql call. The table’s modification timestamp is updated to Utc::now() so subsequent Self::incremental_backup calls can pick it up.

pub fn registered_tables(&self) -> Vec<String>

Returns a snapshot of the table names currently registered, in deterministic (sorted) order.

pub async fn backup( &self, pool_name: &str, output_dir: &Path, ) -> Result<BackupResult, BkError>

Writes a JSON BackupManifest containing every registered table to output_dir. The on-disk file is real - subsequent calls to crate::RestoreManager::restore can read it back.

When config.compress == true the JSON payload is gzip-compressed (magic bytes 0x1f 0x8b) before being written to disk. Restore auto-detects the encoding, so callers do not need to track it.

Source

pub async fn incremental_backup( &self, since: DateTime<Utc>, ) -> Result<BackupManifest, BkError>

Produces an incremental BackupManifest containing only tables whose modified_at >= since. The returned manifest has is_incremental = true and base_backup_id = None; callers can chain BackupManifest::with_base_backup_id to associate it with a full backup.

No file is written - the manifest is returned in-memory so callers can decide where (and whether) to persist it.

Source

pub async fn export_sql( &self, pool_name: &str, output_dir: &Path, ) -> Result<ExportResult, BkError>

Writes a UTF-8 SQL file containing one INSERT statement per registered row. The output is real - it can be re-read by crate::RestoreManager::import_sql.
