### C-2 SQL 注入（MorphTo 关系加载）✅ 已修复

**位置**：[packages/sz-orm-core/src/model.rs:345-348](file:///e:/vue/test/鲜视达/rust/sz-orm/packages/sz-orm-core/src/model.rs#L345-L348)
**现象**：`morph_type_value` 直接作为表名拼接到 SQL，无标识符校验
**证据**：
```rust
let sql = format!(
    "SELECT * FROM {} WHERE id = {}",
    morph_type_value,  // ← 直接拼接，无校验
    pk_to_sql_string(&morph_id_value)
);
```
**影响**：若 `morph_type_value` 来自不可信来源，可构造 SQL 注入
**状态**：✅ 已修复（添加 `is_valid_sql_identifier` 校验）
**修复提交**：本次提交

### C-3 分页 SQL 方言不兼容 ✅ 已修复

**位置**：[packages/sz-orm-core/src/typed_ast.rs:503-508](file:///e:/vue/test/鲜视达/rust/sz-orm/packages/sz-orm-core/src/typed_ast.rs#L503-L508)
**现象**：`TypedSelectQuery::build` 硬编码 `LIMIT/OFFSET`，绕过 `dialect.build_pagination()`
**证据**：
```rust
if let Some(n) = self.limit_n {
    sql.push_str(&format!(" LIMIT {}", n));  // ← 硬编码
}
if let Some(n) = self.offset_n {
    sql.push_str(&format!(" OFFSET {}", n));  // ← 硬编码
}
```
**影响**：Oracle/SQL Server/DB2/ClickHouse 等不支持 `LIMIT/OFFSET` 语法的数据库执行失败
**状态**：✅ 已修复（改用 `dialect.build_pagination()`）
**修复提交**：本次提交

### H-5 real_pulsar.rs ack() 为 no-op

**位置**：[packages/sz-orm-queue/src/real_pulsar.rs](file:///e:/vue/test/鲜视达/rust/sz-orm/packages/sz-orm-queue/src/real_pulsar.rs)
**现象**：Pulsar ack() 为 no-op
**影响**：消息处理后未确认，Pulsar 会重新投递（至少一次语义），但无法显式 nack
**状态**：⚠️ 待修复
**修复建议**：扩展 message_id → consumer 映射，调用 `consumer.ack()`

### H-6 real_nats.rs ack() 为 no-op（NATS Core 限制）

**位置**：[packages/sz-orm-queue/src/real_nats.rs](file:///e:/vue/test/鲜视达/rust/sz-orm/packages/sz-orm-queue/src/real_nats.rs)

### H-8 事务嵌套未限制深度

### H-9 批量插入无分片

**位置**：`packages/sz-orm-batch/src/lib.rs`
**现象**：批量插入可能一次性插入全部记录
