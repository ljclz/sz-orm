# ADR-0004: 批量插入分片防止数据库参数超限

- **状态**: Accepted
- **日期**: 2026-07-19
- **相关代码**: `packages/sz-orm-batch/src/lib.rs` (L73-L84, L120-L129, L239-L247)
- **修复编号**: H-9

## 背景

`batch_insert()` 生成多值 INSERT 语句（`INSERT INTO t (a,b) VALUES (?,?,?), (?,?,?), ...`）。当行数很大时：
- MySQL: 受 `max_allowed_packet` 限制（默认 4MB）
- PostgreSQL: 参数占位符上限 65535（即 65535 / 列数 = 最大行数）
- SQLite: `SQLITE_MAX_VARIABLE_NUMBER` 默认 999（3.32+ 提升至 32766）

不分片会导致超大批量插入失败。

## 决策

在 `DefaultBatchOps` 中加入 `chunk_size: usize`，默认 1000。当 `rows.len() > chunk_size` 时，按 `chunk_size` 分片生成多条独立 INSERT 语句。

```rust
pub const DEFAULT_CHUNK_SIZE: usize = 1000;

fn chunk_indices(&self, total: usize) -> impl Iterator<Item = (usize, usize)> {
    let chunk_size = self.chunk_size.max(1);
    (0..total).step_by(chunk_size).map(move |start| {
        let end = (start + chunk_size).min(total);
        (start, end)
    })
}
```

`batch_insert` 和 `batch_upsert` 分片；`batch_update` 不分片（单条 CASE WHEN 语句）。

## 后果

**正面：**
- 超大批量插入不再触发数据库限制
- 分片大小可配置（`with_chunk_size()`），适配不同数据库

**负面：**
- 多条 INSERT 不是原子操作（除非在事务中执行）
- 分片大小需要调用方根据列数估算（如 10 列 × 6000 行 = 60000 参数，PostgreSQL 安全；但 10 列 × 7000 行 = 70000 > 65535）

**注意事项：**
- `chunk_size = 0` 会被 `.max(1)` 钳制为 1（每行一条 SQL）
- 调用方应在事务中执行分片 INSERT 以保证原子性
