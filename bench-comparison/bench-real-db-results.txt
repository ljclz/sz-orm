=== 综合排名总结 ===

MySQL 远程（5 场景）：
- INSERT 1K:        sz-orm 第一（19.60s）vs sqlx 34.09s, sea-orm 33.86s
- SELECT BY ID:     sz-orm 第一（1.60s） vs sqlx 2.99s, sea-orm 2.71s
- SELECT ALL 1K:    sz-orm 第一（28.22ms） vs sqlx 75.34ms, sea-orm 71.40ms
- UPDATE:           sz-orm 第一（1.96s） vs sqlx 3.08s, sea-orm 2.91s
- DELETE 1K:        sz-orm 第一（19.43s） vs sqlx 30.76s, sea-orm 29.86s
SZ-ORM 在 MySQL 全部 5 场景排名第一

PostgreSQL 本地（5 场景）：
- INSERT 1K:        sz-orm 第一（147.32ms）vs sqlx 166.51ms, sea-orm 167.62ms
- SELECT BY ID:     sea-orm 第一（6.77ms） vs sz-orm 8.28ms, sqlx 8.91ms — SZ-ORM 第二
- SELECT ALL 1K:    sqlx 第一（544.50µs） vs sea-orm 599.60µs, sz-orm 2.34ms — SZ-ORM 第三
- UPDATE:           sea-orm 第一（15.90ms）vs sz-orm 17.51ms, sqlx 19.39ms — SZ-ORM 第二
- DELETE 1K:        sz-orm 第一（137.49ms）vs sea-orm 149.58ms, sqlx 152.23ms
SZ-ORM 在 PG 拿下 INSERT/DELETE 两项第一，SELECT ALL 最弱

环境差异说明：
- MySQL：远程 WAN，网络 RTT ~19ms 主导，轻抽象层优势显现
- PostgreSQL：本机 localhost，网络 RTT ≈ 0，纯抽象层开销对比
- 在无网络干扰的本地公平环境下，SZ-ORM 在 SELECT ALL 落后（format! SQL 构造开销），
  但在 INSERT/DELETE 写场景仍领先
